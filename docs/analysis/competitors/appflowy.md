# AppFlowy Competitor Analysis

## Snapshot
- Tool: AppFlowy
- Analysis date: 2026-02-28
- Positioning: Local-first workspace/databases with self-host options.

## Strengths
- Strong local-first posture and offline-oriented architecture.
- Flexible content/database style interaction model.

## Gaps vs Jira/Asana-Class Capability
- Not primarily a full project operating system with enterprise PM depth.
- Collaboration/operations maturity depends on deployment mode.

## Relevance to Meitheal
- Strong architecture reference for local-first user expectations.

## Implications for Meitheal
- Keep local-first guarantees strong, especially on HA runtime.
- Separate knowledge/config surfaces cleanly (YAML/MD/MDX-driven config + KCS docs).
