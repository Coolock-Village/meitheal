# Plan 02: Security Hardening (SSRF + Rate Limiting)

<objective>
Close the two tracked security concerns: SSRF vulnerability in /api/unfurl and missing rate limiting on compat API routes.
</objective>

<context>
Phase 2 CONTEXT.md decisions (locked):
- SSRF: DNS resolution check (deny private IPs), content-type allowlist, size cap (1MB), timeout (5s)
- Rate limiting: token bucket per IP, 100 req/min default, configurable via env
- Rate limiter at middleware level (consistent with ingress auth pattern)
- Webhook target URLs reuse same SSRF guardrail
</context>

<tasks>
<task id="T-211" priority="P1">
Create `packages/domain-integrations/src/rate-limiter.ts`:
- Token bucket algorithm: `{ capacity, refillRate, refillInterval }`
- In-memory Map<string, Bucket> keyed by IP
- `checkRate(ip: string): { allowed: boolean, remaining: number, resetMs: number }`
- Default: 100 req/min, configurable via `MEITHEAL_RATE_LIMIT_RPM`
- Periodic cleanup of stale buckets (every 5 min)
- Export rate limit headers: `X-RateLimit-Limit`, `X-RateLimit-Remaining`, `X-RateLimit-Reset`
</task>

<task id="T-212" priority="P1" depends="T-211">
Integrate rate limiter into `apps/web/src/middleware.ts`:
- Apply to `/api/v1/*` routes only (compat API)
- Return 429 with `Retry-After` header when exceeded
- Pass-through for non-compat routes
- Structured log on rate limit hit via domain-observability
</task>

<task id="T-213" priority="P1">
Harden `apps/web/src/pages/api/unfurl.ts`:
- DNS resolution check: resolve URL hostname, reject if private IP (10.x, 172.16-31.x, 192.168.x, 127.x, ::1, fc00::/7)
- Content-type allowlist: only fetch text/html, application/json
- Response size cap: 1MB max
- Timeout: 5s via AbortController
- Reuse `isPrivateUrl()` from T-201 (webhook-config.ts)
</task>

<task id="T-214" priority="P1">
Write tests:
- `rate-limiter.spec.ts`: allow within limit, reject over limit, bucket refill, header values
- `ssrf-guard.spec.ts`: reject private IPs (127.0.0.1, 10.0.0.1, 192.168.1.1, ::1), allow public IPs, reject oversized responses
</task>
</tasks>
