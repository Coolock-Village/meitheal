# Plan 01: Webhook Emission Infrastructure

<objective>
Build the core webhook emission system: domain event → HMAC-signed HTTP POST → retry with backoff → dead letter storage. This is the integration backbone that Grocy, n8n, and Node-RED consume.
</objective>

<context>
Phase 2 CONTEXT.md decisions (locked):
- HMAC-SHA256 signing with per-subscriber secrets
- X-Meitheal-Signature header
- Retry: 3 attempts, 1s/4s/16s exponential backoff
- Dead letter queue in SQLite for manual replay
- Config in integrations.yaml (Zod-validated content schema)
- Event payload follows existing DomainEvent<T> envelope
- New event: webhook.delivery.failed
</context>

<tasks>
<task id="T-201" priority="P0">
Create `packages/domain-integrations/src/webhook-config.ts`:
- Zod schema for webhook subscriber config: `{ id, url, secret, events: string[], enabled: boolean }`
- Parse from YAML content schema pattern (match existing framework YAML approach)
- Validate: no private IPs in URL (reuse for SSRF guard in Plan 03)
- Export: `WebhookSubscriberConfig`, `parseWebhookConfig()`, `isPrivateUrl()`
</task>

<task id="T-202" priority="P0" depends="T-201">
Create `packages/domain-integrations/src/webhook-signer.ts`:
- `signPayload(secret: string, payload: string): string` — HMAC-SHA256, hex-encoded
- `verifySignature(secret: string, payload: string, signature: string): boolean`
- Pure function, no side effects, fully testable
</task>

<task id="T-203" priority="P0" depends="T-201,T-202">
Create `packages/domain-integrations/src/webhook-emitter.ts`:
- `emitWebhook(event: DomainEvent<unknown>, subscriber: WebhookSubscriberConfig): Promise<WebhookDeliveryResult>`
- HTTP POST with: `Content-Type: application/json`, `X-Meitheal-Signature`, `X-Meitheal-Event`, `X-Meitheal-Delivery-Id`
- Retry logic: 3 attempts, exponential backoff (1s/4s/16s)
- Timeout: 10s per attempt
- AbortController pattern (matches HA calendar adapter)
- Return: `{ ok, statusCode, attempts, finalError? }`
</task>

<task id="T-204" priority="P0" depends="T-203">
Create dead letter queue:
- Migration `0004_webhook_delivery_log.sql`: table `webhook_deliveries` with columns: id, event_type, subscriber_id, payload, signature, status (delivered|failed|pending_retry), attempts, last_error, created_at, delivered_at
- Drizzle schema addition in `schema.ts`
- Failed deliveries stored after all retries exhausted
</task>

<task id="T-205" priority="P0" depends="T-203,T-204">
Create `apps/web/src/domains/integrations/webhook-dispatch.ts`:
- Listen to domain events bus (existing pattern from task-sync)
- For each event, find matching subscribers from config
- Emit to each subscriber in parallel
- Log delivery results via domain-observability logger
- Store failures in dead letter queue
- Emit `webhook.delivery.failed` domain event on final failure
</task>

<task id="T-206" priority="P1">
Write tests:
- `webhook-signer.spec.ts`: sign/verify round-trip, tamper detection, empty payload
- `webhook-emitter.spec.ts`: successful delivery, retry on 5xx, dead letter on final failure, timeout handling, private IP rejection
</task>
</tasks>
