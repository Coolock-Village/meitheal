# Plan 05-03: PWA + Offline UX Polish

**Wave:** 3
**Scope:** Conflict notification, aria-live sync, precache hashing, apple-touch-icon, manifest screenshots

## Tasks

### T-521: Conflict Notification UI (Audit #14)
- Add `ConflictBanner` component that shows when sync conflict resolved
- Display which task was affected and resolution outcome (server won / local won)
- Auto-dismiss after 8 seconds, manual dismiss available

### T-522: aria-live Sync Region (Audit #25)
- Add `aria-live="polite"` region for sync status announcements
- Screen readers hear: "Syncing 3 changes", "All changes synced", "Sync error: retry in 30s"

### T-523: Precache Revision Hashing (Audit #17)
- Add content hash suffix to precached asset URLs in SW
- Use build-time generated manifest with revision hashes
- Ensures stale assets are never served from cache

### T-524: PWA Metadata (Audits #18, #19, #21)
- Add `apple-touch-icon` meta tag to Layout
- Add iOS splash screen meta tags
- Add `screenshots` array to manifest for Play Store listing

### T-525: Stale Data Age Indicator (Audit #22)
- Show "Last synced: X minutes ago" in UI footer
- Use relative time formatting (`Intl.RelativeTimeFormat`)
- Updates every 30 seconds when visible

### T-526: Tests
- Conflict banner renders on sync.conflict event
- aria-live region updates text on sync state change
- Stale age indicator shows relative time
