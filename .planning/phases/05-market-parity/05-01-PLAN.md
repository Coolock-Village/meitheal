# Plan 05-01: Security Hardening + Auth

**Wave:** 1
**Scope:** timingSafeEqual, CSRF, IP hashing, stale PUT 409, D1 fallback

## Tasks

### T-501: timingSafeEqual for Worker Auth (Audit #1)
- Replace string `===` comparison in `authorizeRequest()` with constant-time compare
- Use `crypto.subtle` (available in Workers) to convert strings to ArrayBuffer and compare
- No timing side-channel for token extraction

### T-502: CSRF Protection (Audit #4)
- Add Origin header validation on all non-GET/HEAD/OPTIONS routes
- Reject requests where Origin doesn't match allowed origins
- Add `SameSite=Strict` guidance for cookie-based auth (future)

### T-503: Stale Update Detection (Audit #7)
- PUT `/api/v1/tasks/:id` accepts `If-Match` header with `updated_at` timestamp
- Return 409 Conflict if stored `updated_at` is newer than client's
- Client receives current task state in error body for merge

### T-504: IP Hashing in Rate Limiter (Audit #5)
- Hash client IP with SHA-256 before using as bucket key
- Prevents IP addresses from being stored in memory as plaintext
- Use `crypto.subtle.digest` (available in Workers)

### T-505: D1 Unavailability Fallback (Audit #39)
- Catch D1 connection errors specifically
- Return 503 with `Retry-After: 30` header
- Log structured error event

### T-506: Tests
- timingSafeEqual: correct token accepted, wrong token rejected, timing consistent
- CSRF: reject missing Origin, reject wrong Origin, allow correct Origin
- 409: reject stale PUT, accept fresh PUT
- IP hashing: hashed key differs from raw IP
- D1 fallback: 503 returned on D1 error

**Total:** 6 tasks
