# Phase 55-01: HA Custom Component — Entity & Action Registration

## Goal

Upgrade the existing Python custom component scaffold to a full integration that
registers todo entities, sensor entities, and custom services with HA Core.

## Architecture

```text
custom_components/meitheal/
├── __init__.py        # Platform loader + service registration
├── config_flow.py     # HA config flow UI (auto-detect Meitheal addon)
├── const.py           # Domain constants, platforms, defaults
├── coordinator.py     # DataUpdateCoordinator — polls Meitheal REST API
├── manifest.json      # config_flow: true, integration_type: hub
├── sensor.py          # SensorEntity: active/overdue/total task counts
├── services.yaml      # Service definitions (add, complete, sync)
├── strings.json       # Config flow + service UI strings
└── todo.py            # TodoListEntity: proxy to Meitheal task CRUD
```

## Communication Pattern

```text
HA Core ←→ custom_components/meitheal ←→ Meitheal addon (REST API)
                                              ↕
                                        SQLite task DB
```

The Python component is a **thin REST proxy** — all task storage stays in
Meitheal's SQLite. The component calls the Meitheal addon REST API at
`http://{ADDON_SLUG}:3000/api/...` using HA's internal Docker network.

## Tasks

### 55-01-01: Create const.py

Extract domain constants into dedicated module:

- DOMAIN = "meitheal"

- DEFAULT_HOST = "local_meitheal_hub" (internal addon hostname)
- DEFAULT_PORT = 3000
- CONF_HOST, CONF_PORT, CONF_API_KEY
- SCAN_INTERVAL = timedelta(seconds=30)

### 55-01-02: Create coordinator.py

DataUpdateCoordinator that polls Meitheal REST API:

- `async_update_data()`: GET /api/tasks → list of tasks
- Parses title, status, due_date, priority, description, id
- Tracks active_count, overdue_count, total_count
- Uses aiohttp.ClientSession with addon base URL
- Error handling with UpdateFailed
- Configurable SCAN_INTERVAL (default 30s)

### 55-01-03: Create todo.py — TodoListEntity

TodoListEntity subclass backed by Meitheal REST API:

- `async_create_todo_item(item)` → POST /api/tasks/create
- `async_update_todo_item(item)` → PUT /api/tasks/{id}
- `async_delete_todo_items(uids)` → DELETE /api/tasks/{id}
- `async_move_todo_item(uid, previous_uid)` → PATCH /api/tasks/reorder
- `todo_items` property → maps coordinator data to TodoItem objects
- Supported features: CREATE | UPDATE | DELETE | MOVE | SET_DUE_DATE | SET_DESCRIPTION
- Status mapping: "todo"/"in_progress" → needs_action, "done" → completed

### 55-01-04: Create sensor.py — Task count sensors

Three SensorEntity subclasses backed by coordinator:

- MeithealActiveTasksSensor: sensor.meitheal_active_tasks
- MeithealOverdueTasksSensor: sensor.meitheal_overdue_tasks
- MeithealTotalTasksSensor: sensor.meitheal_total_tasks

Each uses coordinator.data for fast updates, no extra API calls.

- single_config_entry_only = True (one Meitheal per HA instance)
