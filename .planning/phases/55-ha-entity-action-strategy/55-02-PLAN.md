# Phase 55-02: Addon Auto-Install of Custom Component

## Goal

Eliminate the separate install barrier by having the addon automatically deploy
the custom component to HA Core at startup.

## Strategy

On addon boot, run.sh:

1. Checks if `/homeassistant/custom_components/meitheal/` exists
2. If missing or outdated: copies from `/opt/meitheal/custom_components/meitheal/`
3. Calls HA WebSocket to reload custom components
4. Logs structured event for the install action

The HA config directory is accessible via the `homeassistant_config` map in config.yaml.

## Tasks

### 55-02-01: Update Dockerfile to bundle component

Add COPY step in Dockerfile to include the custom component:

```dockerfile
COPY --from=builder /build/integrations/home-assistant/custom_components/ /opt/meitheal/custom_components/
```

### 55-02-02: Update config.yaml with map

Add `map: ["homeassistant_config:rw"]` to config.yaml so the addon
can write to HA's config directory.

### 55-02-03: Update run.sh with auto-install logic

Add after DB migration but before web server start:
```bash
# Auto-install Meitheal custom component
COMPONENT_SRC="/opt/meitheal/custom_components/meitheal"
COMPONENT_DST="/homeassistant/custom_components/meitheal"
if [ -d "$COMPONENT_SRC" ]; then
  mkdir -p /homeassistant/custom_components
  if [ ! -f "$COMPONENT_DST/manifest.json" ] || \
     ! diff -q "$COMPONENT_SRC/manifest.json" "$COMPONENT_DST/manifest.json" >/dev/null 2>&1; then
    cp -r "$COMPONENT_SRC" "$COMPONENT_DST"
    echo "{\"event\":\"component.installed\",\"path\":\"$COMPONENT_DST\"}"
  fi
fi
```
