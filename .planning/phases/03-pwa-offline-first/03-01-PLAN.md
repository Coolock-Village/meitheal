# Plan 03-01: Service Worker & PWA Manifest

**Wave:** 1 (no dependency on other plans)
**Scope:** Service worker registration, Workbox caching, web app manifest, install prompt

## Tasks

### T-301: Web App Manifest
- Create `public/manifest.webmanifest` with Meitheal branding
- name, short_name, description, start_url, display: standalone
- theme_color, background_color from brand palette
- Icons: 192px + 512px (generate from existing logo or create placeholder)
- Categories: productivity, utilities

### T-302: Service Worker Registration
- Create `src/sw.ts` using Workbox
- Cache-first for static assets (CSS, JS, fonts, images)
- Network-first for API routes (`/api/*`) with offline fallback
- App shell pattern: precache layout skeleton
- Precache manifest budget: ≤ 1MB
- Register SW in Astro layout head

### T-303: Offline Fallback Page
- Create `public/offline.html` — minimal page shown when network unavailable and no cached route
- Must not require JS to render
- Light brand styling, "You're offline" message with retry button

### T-304: Install Prompt Logic
- Defer `beforeinstallprompt` until 2nd visit (localStorage counter)
- Non-intrusive banner component (not modal)
- Dismiss remembers choice for 7 days
- Track install success/dismissal as domain events

### T-305: Service Worker Tests
- SW registration test (mocked navigator.serviceWorker)
- Offline fallback page renders without JS
- Manifest validates against W3C schema

**Total:** 5 tasks
