# Plan 53-01: UX Polish & Warnings

## Objective
Harden the client-side SSE reconnect logic, polish the newly added "Ask HA" conversational modal, and clean up lingering Tailwind CSS build warnings.

## Tasks

1. **[ ] Task 1: Harden Client SSE (`connectivity.ts`)**
   - Add exponential backoff to the SSE `EventSource` reconnection logic to prevent rapid looping if the server is temporarily unavailable or returning 503s (like we saw when the Addon suspended).
   - Ensure the previous `EventSource` is fully closed before initiating a new one.

2. **[ ] Task 2: Polish `AskAssistModal.astro`**
   - Trap focus inside the modal when it is open.
   - Close the modal on the `Escape` key.
   - Auto-focus the input field when the modal opens.
   - Clear the chat history and input field when the modal is closed to provide a fresh state on next open.
   - Utilize the `showToast` utility for network errors instead of appending them as "bot replies" in the chat window, preventing clutter.

3. **[ ] Task 3: Fix Tailwind CSS Warnings (`global.css`)**
   - Identify the cause of the `CssSyntaxError: Tailwind CSS: ...` warnings that appear during `pnpm build`.
   - The warnings are likely due to improper nesting, invalid custom properties, or using Tailwind `@apply` directives inside complex selectors that Tailwind's parser struggles with.
   - Refactor the problematic CSS rules in `global.css` while preserving the exact aesthetic output.
