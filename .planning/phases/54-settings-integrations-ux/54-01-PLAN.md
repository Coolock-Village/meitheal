# Phase 54-01: Settings & Integrations UX Overhaul

## Goal

Transform the Settings General tab and Integrations tab from manual-input, flat-list UX into auto-detecting, visually separated, tooltip-rich, production-quality configuration pages.

## must_haves

- [ ] HA connection auto-detects via SUPERVISOR_TOKEN — no manual URL/token required in addon mode
- [ ] Calendar Entity removed from General tab (only in Integrations)
- [ ] All HA entity inputs replaced with dropdown selectors populated from WebSocket
- [ ] Each integration in its own collapsible card with icon + status badge
- [ ] Help tooltips `(?)` on key labels with popover explanations
- [ ] Todo Sync uses toggle switch instead of raw checkbox
- [ ] Loading skeletons during entity detection
- [ ] Settings.astro JS extracted into domain-specific modules
- [ ] Build passes with zero errors (`pnpm check && pnpm build`)

## Tasks

<task id="54-01-01" title="Rewrite SettingsGeneral.astro — HA Connection Auto-Detect">
Replace the manual HA URL / Long-Lived Token / Calendar Entity form with:
1. A connection status card that auto-detects via `/api/ha/connection`
2. Green/red indicator showing connection mode (addon vs standalone)
3. HA version, location name, timezone display
4. Collapsed "Advanced / Standalone Mode" section with manual URL + Token fields
5. Remove Calendar Entity field entirely (moved to Integrations tab)
</task>

<task id="54-01-02" title="Rewrite SettingsIntegrations.astro — Collapsible Integration Cards">
Split the single flat card into 5 individual collapsible integration cards:
1. Calendar Sync — with EntitySelector dropdown, CalDAV optional field, help tooltips
2. Todo Sync — with EntitySelector dropdown, toggle switch, sync direction
3. Grocy — with URL, API key, sync mode, test connection
4. n8n / Node-RED — with webhook URL, event checkboxes
5. Webhooks — with endpoint, signing secret, payload format

Each card: icon + title + status badge in clickable header, config form on expand.
Collapsed by default unless configured. Standardized badge states.
</task>

<task id="54-01-03" title="Integrate HelpTooltip into Settings Labels">
Add HelpTooltip component next to key labels:
- Calendar Entity: "The HA calendar entity Meitheal syncs with..."
- CalDAV URL: "For direct CalDAV sync with Nextcloud, Radicale..."
- Todo Entity: "The HA todo list entity to sync with..."
- Sync Direction: "Bidirectional keeps both in sync. Inbound-only saves power..."
- Webhook Secret: "HMAC-SHA256 secret for payload signing..."
- n8n Webhook: "Meitheal sends JSON payloads when task events occur..."
</task>

<task id="54-01-04" title="Integrate EntitySelector for Calendar and Todo">
Replace entity text inputs with EntitySelector components:
- Calendar: `<EntitySelector domain="calendar" id="cal-entity" />`
- Todo: `<EntitySelector domain="todo" id="todo-entity" />`
Wire up the save handlers to read from the correct element (select or manual input).
</task>

<task id="54-01-05" title="Extract settings.astro JS into Domain Modules">
Create 3 new TypeScript modules:
1. `settings-integrations.ts` — calendar/todo/grocy/n8n/webhook save/sync/hydrate/restore
2. `settings-general.ts` — HA connection test/save, sidebar config, custom fields
3. `settings-system.ts` — theme, locale, timezone, week start, date format, export/import

Move all corresponding code from the 1972-line `<script>` block in settings.astro.
Each settings tab component imports its own module.
</task>

<task id="54-01-06" title="Consistency Pass — Toggles, Badges, Loading States">
1. Replace Todo Sync checkbox with `.toggle-label` + `.toggle-slider` pattern
2. Standardize all badges to 4 states: Connected, Configured, Not configured, Error
3. Add loading skeleton CSS class `.skeleton-line` for entity detection
4. Add collapsible card CSS classes `.integration-card`, `.integration-card--collapsed`
5. Add field-level validation error styles
</task>
