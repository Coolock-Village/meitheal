# Plan 15-01: UX Parity and Board Domain Separation (Reconstructed)

## Objective

Deliver board-scoped task organization and Trello/Jira-style card interaction parity while preserving Astro-first and HA runtime constraints.

## Workstreams

### WS-1501: Boards as first-class domain boundary

- Add `boards` persistence model and `board_id` task association.
- Auto-create default board and protect core board lifecycle.
- Provide board CRUD API surface.

### WS-1502: Board-scoped UI behavior

- Add sidebar board switcher and persist active board context.
- Apply board filtering consistently across list/kanban/table views.
- Keep board state compatible with existing task payloads.

### WS-1503: Card/task UX parity upgrades

- Add richer kanban card metadata and actions.
- Add RICE filtering and card-level visibility improvements.
- Fix board/custom-field rendering regressions across task views.

### WS-1504: Validation and regression controls

- Ensure API and UI pathways maintain validation and safe fallbacks.
- Verify no DDD boundary regressions and no HA runtime-breaking dependencies.
