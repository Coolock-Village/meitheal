# Phase 3 Plan 1: UI/UX Critical Fixes (P0-P2)

## Objective

Fix the three critical UI/UX regressions: scroll blocking, body overflow stuck, theme inconsistency. Then address P1-P2 improvements.

## Context

- @.planning/phases/03-ui-ux-stabilization/03-RESEARCH.md
- @.planning/codebase/CONCERNS.md

## Tasks

### Task 1: Fix Scroll Blocking
type: auto

**Problem:** `.main-content` has `height: 100vh` + `overflow-y: auto` creating nested scroll context. Body has `flex + min-h-screen + overflow-x: hidden`. Mouse wheel targets body but scrollable element is `.main-content`.

**Changes:**
1. `apps/web/src/styles/_layout.css` — Remove `height: 100vh` from `.main-content`, change to `min-height: 100vh`, remove `overflow-y: auto`
2. `apps/web/src/styles/_base.css` — Keep `overflow-x: hidden` on body but ensure no conflicting overflow rules

**Done when:** Mouse scroll works immediately on page load without clicking first. Verified in browser.

### Task 2: Fix Body Overflow Stuck After Task Detail Close
type: auto

**Problem:** Opening task detail sets `body.style.overflow = 'hidden'`. Escape key close path in `layout-controller.ts` removes panel but does NOT reset overflow. Only `pagehide` resets it.

**Changes:**
1. `apps/web/src/lib/layout-controller.ts` — Add `document.body.style.overflow = ''` in the Escape key handler for task detail overlay
2. `apps/web/src/lib/task-modal-controller.ts` — Ensure `closeTaskDetail()` always resets body overflow

**Done when:** After opening and closing task detail (Escape, click outside, close btn), page scroll works normally.

### Task 3: Fix Theme Init — Explicit data-theme for All States
type: auto

**Problem:** Theme init script only sets `data-theme` for light/auto/custom but not dark. Dark relies on `:root` fallthrough.

**Changes:**
1. `apps/web/src/layouts/Layout.astro` — In theme init inline script, always set `document.documentElement.dataset.theme = theme` regardless of value (dark/light/auto/custom)

**Done when:** `document.documentElement.dataset.theme` is always set on page load. Dark theme shows `data-theme="dark"`.

### Task 4: Remove Duplicate SW Registration
type: auto

**Problem:** SW registered twice — inline block (L1282-1330) and module import (L1451-1497).

**Changes:**
1. `apps/web/src/layouts/Layout.astro` — Remove the inline SW registration block (L1282-1330), keep module import version

**Done when:** Only one `navigator.serviceWorker.register` call in Layout.astro source.

### Task 5: Add Cmd+K Command Palette Handler
type: auto

**Problem:** Command palette exists in DOM but no Ctrl+K/Cmd+K keyboard shortcut to open it.

**Changes:**
1. `apps/web/src/lib/layout-controller.ts` — Add `(e.metaKey || e.ctrlKey) && e.key === 'k'` handler in the keydown listener to toggle command palette visibility

**Done when:** Pressing Cmd+K (Mac) or Ctrl+K (other) opens the command palette overlay.

### Task 6: A11y Improvements
type: auto

**Problem:** Emoji-only buttons have no aria-labels. Task detail panel has no dynamic aria-label.

**Changes:**
1. `apps/web/src/pages/kanban.astro` — Add `aria-label` to edit (✏️), duplicate (⧉), delete (🗑) card action buttons
2. `apps/web/src/layouts/Layout.astro` — Set `aria-label` on task detail dialog dynamically

**Done when:** All interactive emoji buttons have descriptive `aria-label` attributes.

### Task 7: Loading Bar Reduced-Motion Safety
type: auto

**Problem:** Page transition loading bar animation not suppressed by `prefers-reduced-motion`.

**Changes:**
1. `apps/web/src/styles/_tokens.css` — Add `@media (prefers-reduced-motion: reduce)` rule to `.page-loading-bar.active` to disable animation

**Done when:** Loading bar animation is instant (no transition) when reduced motion is preferred.
