# Phase 6 — Wave 2: Task CRUD + List View

## Scope
Full task management: list view with sort/filter, create/edit modals, inline actions, delete confirmation.

## Tasks

### T-611: Task List Page
- **File:** `apps/web/src/pages/tasks.astro`
- Server-rendered task list with client-side interactivity
- Sort by: title, priority, due_date, created_at, status
- Group by: status, priority, due_date (today/this week/later/overdue)
- Search bar for filtering by title
- Empty state with onboarding guidance

### T-612: Create Task Modal
- Client-side modal with form fields: title, description, priority, due_date, status, labels
- Priority selector (1–5 with labels: Critical/High/Medium/Low/Minimal)
- Due date picker (native `<input type="datetime-local">`)
- Posts to `/api/tasks` on submit
- Toast notification on success/error
- Auto-closes and refreshes list

### T-613: Edit Task Inline
- Click task title → inline contenteditable
- Click status badge → cycle through pending → active → complete
- Click priority badge → dropdown picker
- PUT to `/api/tasks/[id]` on change
- Optimistic UI update

### T-614: Delete Task
- Swipe/hover reveals delete button
- Confirmation dialog before deletion
- DELETE to `/api/tasks/[id]`
- Toast notification
- Animated removal from list

### T-615: Quick-Add Bar
- Persistent input at top of task list
- Enter to create task (title only, defaults to pending/medium priority)
- Shift+Enter for full modal create
- Focus with keyboard shortcut `n`

## Verification
