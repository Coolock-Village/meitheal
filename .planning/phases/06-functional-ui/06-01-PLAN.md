# Phase 6 — Wave 1: Design System + Layout + Dashboard

## Scope
Foundation layer: global CSS design system, Layout.astro component, dashboard page with live stats.

## Tasks

### T-601: Global CSS Design System
- **File:** `apps/web/public/styles/global.css`
- Dark theme: deep navy (`#0f0f1a`) + emerald green (`#10b981`)
- Inter font, CSS custom properties for all tokens
- Components: cards, buttons, forms, modals, toasts, badges, empty states
- Task-specific: checkbox, task-item, kanban-column, table styles
- Responsive breakpoints: 768px, 480px

### T-602: Layout Component
- **File:** `apps/web/src/layouts/Layout.astro`
- Sidebar: logo (☘), nav links (Dashboard, Tasks, Kanban, Labels, Settings)
- Health status indicator (polls `/api/health` every 30s)
- Mobile hamburger menu
- Active page highlighting
- Topbar with page title + action slot

### T-603: Dashboard Page
- **File:** `apps/web/src/pages/index.astro`
- Stats grid: total tasks, active, completed, overdue
- Recent tasks widget (5 most recent)
- Calendar sync status summary
- HA connection status card
- Quick-add task inline form
- All data fetched via Astro SSR from persistence layer

### T-604: API Routes for UI
- **File:** `apps/web/src/pages/api/tasks/index.ts` (GET all, POST create)
- **File:** `apps/web/src/pages/api/tasks/[id].ts` (GET one, PUT update, DELETE)
- Use SQLite/libsql persistence via existing `ensureSchema` + store
- Return JSON with proper error handling
- Title validation (400 on empty)
