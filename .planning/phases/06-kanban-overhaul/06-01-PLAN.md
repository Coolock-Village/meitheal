  <files>apps/web/src/pages/kanban.astro</files>
  <action>
    Apply 3 targeted edits to kanban.astro:

    1. **Remove full name text from user filter pills** (~line 1798-1800):
       Remove the `<span>` element that renders `u.display_name || u.name || u.id` as visible text.
       The initials avatar circle (`btn > span.avatar`) already exists, and `btn.title` provides
       the full name on hover. Add a comment explaining the Teamhood/Jira/Linear pattern.

    2. **Remove board-level tabindex** (~line 1403-1404):
       Replace `board.setAttribute("tabindex", "0")` with a comment explaining why it was removed.
       Board-level tabindex causes drag ghost artifacts and makes the entire board element
       draggable when users click empty space. Keyboard navigation still works via individual
       card `tabindex`/`focus` event handlers.

    3. **Resolve board titles in group-by-board handler** (~line 1554+):
       - Convert `groupBySelect?.addEventListener("change", () => {` to `async () => {`
       - After saving to localStorage, add board title resolution:
         ```javascript
         let boardTitleMap = {};
         if (mode === "board") {
           try {
             const res = await fetch((window.__ingress_path || "") + "/api/boards");
             if (res.ok) {
               const data = await res.json();
               (data.boards || []).forEach(b => {
                 boardTitleMap[b.id] = `${b.icon || '📋'} ${b.title}`;
               });
             }
           } catch { /* use raw IDs as fallback */ }
           boardTitleMap["default"] = "📋 Default";
         }
         ```
       - At ~line 1673, change the label resolution to use `boardTitleMap`:
         `icons[key] || (mode === "board" ? (boardTitleMap[key] || \`📋 ${key}\`) : defaultLabel)`

    **Important:** Use `window.__ingress_path` for the fetch URL to ensure HA ingress compatibility.
    The `Layout.astro` monkey-patches `window.fetch` for ingress, but explicit prefix is safer here
    since this is an inline script.
  </action>

    Add CSS constraints to enable vertical scrolling within Kanban columns:

    1. **`.kanban-board`**: Add `max-height: calc(100vh - 200px)` to constrain the overall board

       Meitheal toolbar). Keep existing properties unchanged.

    2. **`.kanban-column`**: Add `max-height: 100%` so columns inherit the board's height constraint.
