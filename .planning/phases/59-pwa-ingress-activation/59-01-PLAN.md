# Plan 59-01: PWA Ingress Activation

## Objective

Wire the existing but disconnected PWA infrastructure so the service worker registers, the install prompt works, and the update flow is complete — all while respecting HA ingress constraints.

## Context

See `59-CONTEXT.md` and `.planning/persona-loops/pwa-ingress-50-persona-audit.md`.

**Root cause:** `registerServiceWorker()` exported from `sw-register.ts` but never called.

## Tasks

<task id="01" file="apps/web/src/layouts/Layout.astro">
### Wire SW Registration + Install Prompt + Update Handler

Add before `</body>` in Layout.astro:

1. **Module script** that imports `registerServiceWorker` and `activateWaitingSw` from `@domains/offline/sw-register`
2. On DOMContentLoaded: call `registerServiceWorker()`
3. `beforeinstallprompt` listener → stores event on `window.__pwa_install_prompt`, dispatches `meitheal-pwa-installable` custom event
4. `appinstalled` listener → dispatches `meitheal-pwa-installed`, clears stored prompt
5. `meitheal-sw-update` listener (from sw-register.ts) → shows toast with "Update available — Refresh" button, on click calls `activateWaitingSw()` then reloads

**Verify:** Console shows `[sw-register]` messages, no errors.
</task>

<task id="02" file="apps/web/src/layouts/Layout.astro">
### Fix Meta Tags

1. Change `theme-color` from `#1a1a2e` to `#0f172a` (match manifest)
2. Add `<meta name="apple-mobile-web-app-capable" content="yes" />`
3. Add `<meta name="apple-mobile-web-app-status-bar-style" content="black-translucent" />`

**Verify:** View page source, all meta tags present.
</task>

<task id="03" file="apps/web/public/sw.js">
### Update SW Version + Add Version Query

1. Change `CACHE_VERSION` from `"0.1.25"` to `"0.1.58"`
2. Add `GET_VERSION` message handler that responds with `{ type: "VERSION", version: CACHE_VERSION }`

**Verify:** SW activates with new cache name.
</task>

<task id="04" file="apps/web/src/domains/offline/sw-register.ts">
### Improve Ingress Path Delivery

1. After `register()`, use `navigator.serviceWorker.ready` to await active SW before sending `SET_INGRESS_PATH`
2. Keep existing fallback sends to `installing`/`waiting` SWs

**Verify:** No race condition on first registration.
</task>

<task id="05" file="apps/web/src/pages/manifest.webmanifest.ts">
### Stabilize Manifest ID

1. Change `id: prefix("/")` to `id: "/"` (stable identity, not tied to ingress token)
2. Add `lang: "en"` to `MANIFEST_BASE`

**Verify:** Manifest JSON has `id: "/"` regardless of ingress state.
</task>

<task id="06" file="apps/web/src/components/pwa/PwaInstallBanner.astro">
### Create PWA Install Banner Component

Create a new Astro component:
- Fixed bottom banner with glass effect
- "Install Meitheal for offline access" + Install button + dismiss (X)
- Listens for `meitheal-pwa-installable` event → shows banner
- On install click: calls `window.__pwa_install_prompt.prompt()`
- On dismiss: hides + sets `localStorage.meitheal-pwa-dismiss` with 7-day expiry
- Listens for `meitheal-pwa-installed` → hides permanently
- Does not show if already dismissed within 7 days

**Verify:** Banner appears on localhost, dismissal persists.
</task>

Import and render `PwaInstallBanner` before the closing `</body>`.

**Verify:** Banner HTML present in page source.
</task>
