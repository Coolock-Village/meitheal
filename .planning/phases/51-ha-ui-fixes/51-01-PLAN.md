# Phase 51 Plan — HA UI Resiliency

## Implementation Steps

1. **Manifest Authorization Fix**
   - Modify `<link rel="manifest">` in `Layout.astro` to include `crossorigin="use-credentials"`.
   - This ensures the HA Ingress session cookie is sent when fetching the manifest, preventing the 401 Unauthorized error.

2. **UI "Loafing" (Unresponsiveness) Fixes**
   - Review global button states (`New Task` modal and other `btn-primary` elements).
   - Ensure the `isLoading` state in modals/forms is always reset in a `finally {}` block.
   - Specifically check `NewTaskModal.astro` for async form submission bugs that leave the button in a loading state if the `/api/tasks` endpoint hangs or returns 503.

3. **Fetch Interceptor Resilience**
   - Modify the global `fetch` interceptor in `Layout.astro` to implement a timeout on API requests or gracefully catch 503s.
   - If an API request fails with 503 or a network error, it should trigger an error toast and reject rather than hanging indefinitely.

## Verification
- Run a local build: `pnpm --filter @meitheal/web build`
- Verify the manifest link has `crossorigin="use-credentials"`.
- Verify the UI gracefully handles simulated offline states.
