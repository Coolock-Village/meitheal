# Plan 18-01: Vikunja Card Parity and Task Detail UX (Reconstructed)

## Objective

Close major task-detail parity gaps vs Vikunja with a richer task panel, comments support, and command-palette-assisted workflows.

## Workstreams

### WS-1801: Schema and API expansion

- Extend task schema with detail-oriented fields (dates, progress, color, favorite).
- Add comments persistence/API surface tied to task detail.

### WS-1802: Task detail panel and deep-linking

- Implement slide-out task detail panel in app layout.
- Support hash/click entry points and robust open/close behavior.

### WS-1803: Command palette and quick actions

- Add keyboard-triggered command palette (`Ctrl+K` / `Cmd+K`).
- Provide task/board quick actions.

### WS-1804: Regression fixing and roadmap closure

- Fix click-to-open integration regressions after first implementation.
- Record phase status updates in roadmap docs.
