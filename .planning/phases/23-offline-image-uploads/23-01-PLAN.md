# Phase 23: Offline Image Attachments

## Status: Draft

## Objective

Enable users to attach images to tasks while completely offline, persisting the blob data to IndexedDB, and rendering thumbnails on the task cards and task detail overlays.

## Scope

- **Storage Layer:** Extend `offline-store.ts` to support storing `Blob` or Base64 payload representations of images associated with a `taskId`. Will require a new object store `task_attachments`.
- **UI Integration:** Add an "Attach Image 🖼️" button to the Task Detail panel (`Layout.astro`).
- **Display:** Render attached images as thumbnails within the Kanban board cards (`kanban.astro`) and within the Task Detail overlay.
- **Portability:** Ensure the JSON export (completed in Phase 21) handles base64 encoding of image attachments for true sovereign portability.

## Verification

- Images can be attached via file picker.
- Images persist across page reloads (offline IDB storage).
- Image thumbnails appear on Kanban boards.
- Images can be deleted.
