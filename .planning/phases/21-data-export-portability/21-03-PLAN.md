# Phase 21-03: User Preferences and Settings Portability

## Status: Draft

## Objective

Ensure that not just data, but system configuration (Home Assistant connection strings, custom tags, WIP limits, layout toggles) can be exported and imported seamlessly (R-513).

## Scope

- **Settings Payload:** Consolidate all keys currently stored in `localStorage` or non-task `IndexedDB` tables into a unified JSON format (e.g., `meitheal-settings-v1.json`).
- **Export Action:** Add "Export Settings" button to the Settings panel.
- **Import Action (Restore):** Add "Restore Settings" option with a file `<input type="file" accept=".json">`.
- **Validation (Import):** Use Zod or manual type checking to safely parse uploaded settings config, preventing broken UI states if a user uploads a malformed file.
- **Application Reload:** Trigger an optimistic UI update or an intentional page reload after successfully restoring settings to ensure the HA client re-initializes with the new token.

## Verification

- Export a settings file, mutate a value in LocalStorage (like HA token), then restore the file to verify the original state is reset.
