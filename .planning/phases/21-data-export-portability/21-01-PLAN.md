# Phase 21-01: CSV and JSON Task Export

## Status: Draft

## Objective

Implement a robust, client-side data export utility that allows users to download their entire task library as either a raw JSON array or a parsed CSV file for external analysis, satisfying requirement R-511.

## Scope

- **Export Service:** Create an `export-service.ts` in the `offline` or `tasks` domain to fetch all tasks from the local SQLite/IndexedDB store.
- **JSON Generation:** Serialize the full task array, including all custom fields and metadata, into a formatted JSON Blob.
- **CSV Generation:** Map core task properties (ID, Title, Status, Priority, Type, Dates) to a flat CSV structure, properly escaping commas and quotes in titles/descriptions.
- **UI Integration:** Add "Export to JSON" and "Export to CSV" buttons to the `settings.astro` page.
- **Download Trigger:** Use a temporary `<a>` tag with a dynamically generated `Blob` URL to trigger the Native download prompt.

## Out of Scope

- Importing CSV data back into the system (handled separately if requested).
- Cloud-side server exports (this is a local-first guarantee; the client DB is the source of truth).

## Verification

- Verify JSON export contains all known fields and valid types.
- Verify CSV export opens correctly in a standard spreadsheet application without row corruption.
