# Phase 21-02: SQLite Direct Database Download

## Status: Draft

## Objective

Provide ultimate data sovereignty by allowing users to download the raw SQLite/IndexedDB persistence layer directly from their browser, satisfying requirement R-512.

## Scope

- **OPFS / IndexedDB Extraction:** Implement a secure read stream that extracts the underlying `wa-sqlite` binary database file from the browser's OPFS (Origin Private File System) or IndexedDB store (depending on environment constraints/Safari parity).
- **Binary Blob Construction:** Ensure the file is not corrupted during extraction and is served as a precise `application/x-sqlite3` blob.
- **UI Integration:** Add a "Download Raw Database" button to the `settings.astro` page inside the Data Portability section.
- **Documentation:** Add inline tooltips explaining that this is the primary, unmodified data store of their local-first application.

## Verification

- Download the raw `.sqlite` or `.db` file.
- Open it using a standard SQLite viewer (e.g., `sqlite3`, DBeaver) and query the `tasks` table successfully without corruption errors.
