# Phase 52 Plan — HA Native Notifications & Assist

## Implementation Steps

1. **HA Service Calls for Notifications**
   - Update `ha-services.ts` to expose `notifyDevice(message, data)`.
   - Update `/api/tasks` POST/PUT routes: If a task with `priority=1` (Urgent) is created, automatically fire a native push notification to `notify.notify` (which alerts all companion apps) with actionable buttons: "Mark Done" and "View".

2. **Actionable Notification Callback (Webhook/Event bridging)**
   - Modify `ha-connection.ts` to subscribe to the `mobile_app_notification_action` event.
   - When an event is received where `action` matches `MEITHEAL_TASK_DONE_<uuid>`, the backend will update the DB locally without needing the frontend.

3. **HA Assist / Conversation UI integration**
   - Implement `askAssist(text)` in `ha-services.ts` wrapping the `conversation.process` service call.
   - Add an SSE stream to proxy the Assist reply back to the frontend.
   - Create a global `VoiceCommand.astro` floating action button (FAB) or integrate it into the TopNavigation header that allows users to ask HA to turn on lights/run scripts right from Meitheal.

## Verification
- Test `notifyDevice` by creating an Urgent task and watching `ha-system` logs.
- Test `mobile_app_notification_action` subscription by firing a fake event across the WS bus.
- Verify the conversation endpoint returns text.
