# Plan 43-01: Security & Resilience Hardening

## Objective

Address 8 remaining medium-severity persona audit findings (2 already resolved in Phase 44).

## Tasks

<task name="timingSafeEqual_auth" file="apps/web/src/middleware.ts">
Replace string comparison for bearer token auth with `crypto.timingSafeEqual()` to prevent timing attacks.
</task>

<task name="csrf_tokens" file="apps/web/src/middleware.ts">
Implement per-session CSRF tokens: server generates token → injected into page → client sends in X-CSRF-Token header → server validates on POST/PUT/DELETE.
</task>

<task name="optimistic_locking" file="apps/web/src/pages/api/tasks/[id].ts">
Add optimistic locking: PUT/PATCH requires `updated_at` in body, compare against DB value, return 409 if stale with `{ error: "Conflict", server_updated_at: "..." }`.
</task>

<task name="sync_queue_ttl" file="apps/web/src/domains/offline/offline-store.ts">
Add TTL cleanup: on `initOfflineStore()`, delete sync queue entries older than 7 days. Run on every init.
</task>

<task name="conflict_toast_ui" file="apps/web/src/layouts/Layout.astro">
Add conflict notification toast: listen for 409 responses in fetch interceptor → show "Task was modified by another user. Refresh?" toast.
</task>

<task name="d1_fallback" file="apps/web/src/pages/api/tasks/index.ts">
Wrap SQLite operations in try/catch. On failure, return 503 with `{ error: "Database unavailable", cached: true }` and attempt to serve from in-memory cache if available.
</task>

<task name="aria_live_sync" file="apps/web/src/layouts/Layout.astro">
Add `aria-live="polite"` region that announces sync status changes (syncing, synced, conflict, offline).
</task>

<task name="sw_content_hash" file="apps/web/public/sw.js">
Add content hash to SW precache URLs to bust stale caches on deploy.
</task>

## Verification

- Build passes
- `curl` with wrong bearer token rejected in constant time
- PUT with stale `updated_at` returns 409
- Browser console shows CSRF token in meta tag
- Sync queue older entries get cleaned up
